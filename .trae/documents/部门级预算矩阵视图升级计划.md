# 实施计划：部门级预算功能升级

根据您的要求，我将对 `BudgetTransfer.vue` 中的“部门级预算” (Tab 1) 进行深度改造。

## 1. 核心数据结构升级
设计支持三级结构（上级部门 -> 下级部门 -> 预算科目）的数据模型。

```typescript
// 预算科目节点
interface BudgetCategory {
  key: string
  name: string // "加班费", "离职补偿金"
  totalBudget: number
  usedAmount: number
  remaining: number // computed: total - used
  transferAmount: number // 用户输入的划转金额
}

// 下级部门节点
interface SubDeptNode {
  key: string
  name: string // "产品组"
  categories: BudgetCategory[]
}

// 上级部门（当前选择的部门）
interface DeptMatrix {
  key: string
  name: string // "青云工作室"
  subDepts: SubDeptNode[]
}
```

## 2. 界面改造：矩阵视图 (Matrix View)
替换原有的“左中右”三栏布局，改为**左右对照的树形表格 (Tree Table)** 结构。

### 2.1 转出方矩阵 (Source Matrix) - 左侧
*   使用 `a-table` 展示层级数据。
*   **列定义**：
    *   `部门/科目` (树形展开)
    *   `已使用` (实时数据)
    *   `总预算` (初始值)
    *   `余额` (高亮显示，作为划转上限)
*   **汇总逻辑**：上级部门行自动汇总下级部门的所有数据。

### 2.2 接收方矩阵 (Target Matrix) - 右侧
*   结构与左侧保持一致，展示接收部门的层级。
*   **关键列：划转金额 (Action)**
    *   在最末级的“预算科目”行显示 `a-input-number`。
    *   支持用户输入金额。
    *   **实时校验**：
        *   `min="0"`
        *   `max="转出方对应科目的余额"`
        *   若输入金额 > 转出方余额，输入框变红，并显示 Error Tip。

## 3. 业务逻辑实现
*   **联动选择**：
    *   当 Header 选择了 `sourceDeptId`，左侧加载该部门的矩阵数据。
    *   当 Header 选择了 `targetDeptId`，右侧加载该部门的矩阵数据。
*   **自动计算**：
    *   **汇总行计算**：前端实时计算上级部门的 Total 值。
    *   **总划转金额**：实时累加所有输入框的值，更新到底部 Footer。
*   **超额校验**：
    *   提交时遍历所有科目，存在 `transferAmount > remaining` 则阻断。

## 4. 数据持久化 (模拟)
*   增加 `OperationLog` 结构。
*   提交成功后，打印完整的划转日志 (Transaction Record)，并模拟扣减余额。

## 5. 开发步骤
1.  **定义 Mock 数据**：生成包含三级结构的测试数据 (青云工作室 -> 产品组 -> 加班费)。
2.  **重构 Template**：将 Tab 1 的内容替换为左右分栏的 `a-table`。
3.  **实现计算逻辑**：编写 `computed` 或 `watch` 逻辑处理汇总和校验。
4.  **样式调整**：优化表格紧凑度，确保在单屏内展示清晰。
