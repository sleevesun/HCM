# 实施计划：部门级预算表格重构

根据您的新需求，我将对 `BudgetTransfer.vue` 中的“部门级预算”表格进行彻底重构。

## 1. 数据结构改造
从“三级树状结构”变更为“二级扁平结构 + 动态列”。

### 1.1 数据模型
```typescript
interface DeptBudgetRow {
  key: string
  name: string // 部门名称
  isSummary: boolean // 是否为总计行
  // 动态列数据：Key为预算科目ID
  overtime: BudgetDetail // 加班费
  severance: BudgetDetail // 离职补偿金
  signing: BudgetDetail // 签约金
  adjustment: BudgetDetail // 调薪
}

interface BudgetDetail {
  total: number // 总预算
  used: number // 已使用
  remaining: number // 余额 (total - used)
  transferAmount?: number // 划转金额 (仅 Target 表)
}
```

### 1.2 固定预算科目
定义常量 `FIXED_CATEGORIES`：
*   `overtime`: 加班费
*   `severance`: 离职补偿金
*   `signing`: 签约金
*   `adjustment`: 调薪

## 2. 界面改造 (UI)

### 2.1 表格列定义
*   **固定列**：`部门` (width: 150px)
*   **动态列**：遍历 `FIXED_CATEGORIES` 生成 4 列。
    *   表头：科目名称。
    *   单元格内容：
        *   **Source 表**：显示 `remaining`。
            *   样式：`>0` 黑色，`<0` 红色。
            *   交互：`a-popover` 悬停显示明细 (已使用、总预算、公式)。
        *   **Target 表**：显示 `transferAmount` 输入框。
            *   输入框下方显示 `remaining` 预览 (接收后余额)。

### 2.2 总计行 (Summary Row)
*   作为表格的第一行 (`unshift` 到数据源)。
*   `name` 显示为 "总计"。
*   各列数据为所有子部门对应科目的累加值。
*   样式加粗，背景色区分。

## 3. 业务逻辑实现

### 3.1 Mock 数据生成器升级
*   修改 `generateMatrixData`，不再生成三级 `children`，而是直接生成包含 4 个固定科目数据的部门行。
*   生成数据时，随机产生一些负数余额以测试警示样式。

### 3.2 自动计算
*   **总计行计算**：在生成数据后，立即遍历计算总和并插入第一行。
*   **输入校验**：
    *   输入金额必须 `<= Source对应部门对应科目的余额`。
    *   若 Source 余额为负，禁止划转。

## 4. 开发步骤
1.  **重构 TypeScript 接口**：定义新的 `DeptBudgetRow` 结构。
2.  **更新 Mock 生成逻辑**：适配新的 4 个固定科目。
3.  **重写 `a-table` 列配置**：使用 `v-for` 生成动态列。
4.  **实现 Hover 明细**：使用 `a-popover` 组件展示详细数据。
5.  **调整样式**：实现正黑负红、总计行高亮。
