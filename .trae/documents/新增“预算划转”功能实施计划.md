# 实施计划：新增"预算划转"功能

根据《组织资源划转功能设计规范书》及现有项目架构，制定以下开发计划：

## 1. 新建视图组件
创建文件 `src/views/BudgetTransfer.vue`，实现完整的业务逻辑和界面交互。

### 1.1 界面布局 (UI)
*   **整体结构**：采用通栏卡片式布局，分为 Header (基础配置)、Body (核心操作)、Footer (底部操作栏)。
*   **Header 区域**：
    *   转出/接收部门选择器 (使用 `a-select`，模拟数据加载)。
    *   生效日期 (使用 `a-date-picker`)。
    *   划转原因 (使用 `a-textarea`)。
*   **Body 区域 (Tabs)**：
    *   **Tab 1: 部门级预算**
        *   左侧：资产看板 (展示余额)。
        *   中间：动作区 (金额输入、全部划转按钮)。
        *   右侧：模拟预览 (计算转后余额)。
    *   **Tab 2: 空缺 HC**
        *   使用穿梭框 (Shuttle) 或左右双表格布局。
        *   左侧：待划转 HC 池 (支持筛选)。
        *   右侧：拟划转清单。
    *   **Tab 3: 补差额 HC 预算**
        *   列表映射视图，每行展示 Source 余额 -> 输入框 -> Target 预览。
*   **Footer 区域**：
    *   实时汇总信息 (预算金额 + HC 数量)。
    *   操作按钮：取消、保存草稿、提交审批。

### 1.2 交互逻辑 (Logic)
*   **状态管理**：使用 Vue `ref/reactive` 管理表单数据。
*   **校验规则**：
    *   转出/接收部门互斥校验。
    *   非实体部门阻断 (Tab 2/3 禁用)。
    *   金额超支校验 (输入值 <= 余额)。
*   **数据模拟**：
    *   模拟部门列表数据。
    *   模拟 HC 列表数据。
    *   实现前端计算逻辑 (实时更新余额预览)。

## 2. 更新路由配置
修改 `src/router/index.ts`：
*   引入 `BudgetTransfer.vue` 组件。
*   新增路由 `/budget-transfer`，命名为 `BudgetTransfer`。

## 3. 更新导航菜单
修改 `src/views/Dashboard.vue`：
*   在“编制管理”子菜单下新增 `key="3-3"` 的菜单项 "预算划转"。
*   更新 `handleMenuClick` 方法，处理路由跳转。
*   更新路由监听逻辑，确保菜单高亮状态正确。

## 4. 验证测试
*   验证菜单跳转是否流畅。
*   验证页面布局是否符合 PRD 视觉规范。
*   测试核心校验逻辑 (如选择相同部门应报错、金额超支应变红)。
