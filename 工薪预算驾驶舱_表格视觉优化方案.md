# 工薪预算驾驶舱表格视觉优化方案

针对「当前工薪执行现状及现行规划」模块中由于指标繁多、数据密集导致的信息过载问题，在**不修改现有表头**的前提下，提供以下三种视觉与交互优化方案。旨在通过格式化、层级划分和可视化手段，降低认知负荷，提升企业用户的阅读与决策效率。

---

## 方案一：基于业务模块的「列背景色块分组」 (Column Background Grouping)

### 设计原理
运用格式塔心理学中的**相似性（Similarity）**与**封闭性（Enclosure）**原则。由于“执行现状”与“现行规划”是两个不同的业务维度，通过为这两大列组赋予不同的、极低饱和度的背景色，可以在视觉上形成天然的区块隔离，引导用户视线按区块纵向阅读，避免数据在横向对比时产生串行和混淆。

### 实施步骤
1. **色彩定义**：为“执行现状”相关列设置极浅的冷色调背景（如极浅蓝 `#F5F7FA`），为“现行规划”相关列设置极浅的暖色调或中性色背景（如极浅灰 `#FAFAFA`），保持文字对比度符合 W3C 无障碍标准。
2. **斑马线弱化**：如果表格原有强烈的行斑马线，需调低其对比度，或改为仅在鼠标悬浮（Hover）时显示行高亮，避免与列背景色产生视觉冲突，形成“棋盘”错觉。
3. **边界处理**：在两个大模块的分界线处，加深或加粗垂直分割线（如 `1px solid #DCDFE6`），强化业务边界感。

### 预期效果
用户一眼即可区分出当前看的是“现状”还是“规划”，大面积的柔和色块能在不增加额外视觉噪音的情况下，大幅减少在宽表格中横向对比时产生的迷失感。

### 适用场景
适用于具有明确业务分类的宽表格，尤其是包含多个大类、每个大类下又有多个子指标的复杂数据透视表。

### 评估
- **技术实现难度**：**低**。前端仅需通过 CSS 的 `nth-child` 伪类或给特定列添加 `className` 来设置 `background-color`。
- **对现有系统影响**：**极小**。纯样式层面的调整，不涉及底层数据结构和组件逻辑的重构。

## 方案二：基于数据权重的「排版与字重层级」 (Typography & Font Weight Hierarchy)

### 设计原理
企业级应用中并非所有数据都同等重要。通过**视觉层级（Visual Hierarchy）**，让核心数据（如总计、差异绝对值）在视觉上“跳”出来，而次要数据（如占比、历史同期）则适度后退，从而缓解满屏相同粗细的数字带来的压迫感和认知疲劳。

### 实施步骤
1. **主次字重对比**：将关键列（如“本月执行总额”、“规划总额”）的数字字重加粗（`font-weight: 600` 或 `500`），颜色使用主要文本色（如 `#303133`）。
2. **弱化辅助数据**：将衍生指标（如“环比增幅”、“占预算比”）字号调小 1-2px，或使用次级文本色（如中灰色 `#909399`）。
3. **对齐规范优化**：确保所有金额与百分比严格**按小数点对齐（Tabular Figures）**，千位分隔符保持一致。利用数字自身的长短形成自然的留白节奏，而非机械的居中对齐。

### 预期效果
信息绝对密度并未减少，但用户的视觉焦点会被自动引导至核心指标上，过滤掉次要噪音，提升扫视（Scanning）效率。

### 适用场景
适用于包含大量计算衍生指标（如环比、同比、完成率、差异值）的财务或人力薪酬报表，用户通常只需重点关注几个核心金额。

### 评估
- **技术实现难度**：**低**。前端通过配置表格列属性（Column Render）修改特定列的字体样式即可。
- **对现有系统影响**：**小**。需要 UI/产品明确划分指标的主次优先级，前端按规则配置渲染逻辑即可。

## 方案三：数据条与条件格式化「微型可视化」 (Micro-charts & Conditional Formatting)

### 设计原理
利用**前注意视觉处理（Preattentive Processing）**机制，人脑处理颜色和图形的速度远快于阅读纯文本数字。将部分枯燥的数字转化为直观的微型图形，可以极大地压缩信息提取的时间，让数据自我表达。

### 实施步骤
1. **进度/占比可视化**：对于“执行进度”、“预算使用率”等百分比列，在单元格数字背后或下方叠加浅色的 CSS 数据条（Data Bar），条形长度与百分比严格挂钩。
2. **红绿条件格式**：对于“超支额”或“差异率”等预警性质的数据，应用红绿双色（如正向增益用绿色，负向超支用红色），并辅以方向箭头（↑/↓）或预警 Icon。
3. **数字降噪（可选）**：配合图形展示，可考虑将过长的大额数字进行单位换算（如将 15,230,000 简化展示为 1,523 万，鼠标 Hover 时显示精确值）。

### 预期效果
表格不再是死板的数字堆砌，而是变成了生动的数据仪表盘。用户无需逐个阅读数字，仅通过图形长短和颜色即可瞬间定位到异常（如预算超支的红色高亮项），极大增强了界面的美观性和专业度。

### 适用场景
驾驶舱（Cockpit）或 Landing Page 这种强调“全局概览”、“监控”和“异常发现”的业务场景。

### 评估
- **技术实现难度**：**中**。需要前端在表格单元格内自定义渲染（Custom Cell Render），写入包含简单 DOM 结构的数据条逻辑或引入轻量级微图表库。
- **对现有系统影响**：**中**。如果是极大数据量且未做虚拟滚动（Virtual Scrolling）的表格，过多的自定义 DOM 节点可能轻微影响渲染性能；需评估前端现有 Table 组件的扩展能力。

---

## 总结与落地建议

作为 B2B 企业级系统，效率与清晰度是第一位的。在不改动表头的前提下：
- 如果追求**最快落地和最低风险**，建议组合使用 **方案一（列背景分组）** 和 **方案二（排版层级）**，这能在较小的开发成本下显著提升表格质感与可读性。
- 如果追求**最佳的驾驶舱监控体验**，建议引入 **方案三（微型可视化）**，将关键的规划差异和执行率图形化，这最符合 Landing Page 快速发现业务异常的产品定位。
