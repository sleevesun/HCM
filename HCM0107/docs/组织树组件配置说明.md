# 组织树组件配置说明

## 数据源
- 预设脚本：`scripts/initOrgArchitecture.ts`
- 数据存储：`localStorage.org_architecture_db_v1`
- 数据结构定义：`src/mocks/orgData.ts`

## 组织编码与路径
- 组件输出值：`code`
- 组件 `change` 事件附带：
  - `code`：组织编码
  - `name`：组织名称
  - `path`：组织路径（`/` 分隔）

## 动态加载
- 首屏仅加载一级组织树节点。
- 展开节点时通过 `fetchOrgChildren(parentCode)` 按需拉取子节点。

## 搜索筛选
- 使用 `searchText` 聚合字段匹配，包含：
  - 组织名称
  - 组织编码
  - 负责人
  - 人员数量

## 权限控制
- 默认基于当前用户部门编码 `getCurrentUserDeptCode()` 计算可见/可选范围。
- 可通过 `userDeptCode` 覆盖默认权限域。
- 不在权限范围的节点标记为禁用并显示“无权限”。

## 维护接口
- `createOrgNode(payload)`：新增组织
- `updateOrgNode(code, patch)`：更新组织
- `deleteOrgNode(code)`：删除组织（含级联子节点）
- `validateOrgArchitecture(records)`：校验结构一致性
