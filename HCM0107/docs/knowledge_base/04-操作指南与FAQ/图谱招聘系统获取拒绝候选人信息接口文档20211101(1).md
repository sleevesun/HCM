图谱招聘系统获取拒绝候选人信息接口文档

前提条件
访问职位接口的服务器IP需加入白名单许可，请确认当前调用的机器为约定IP，并可访问
Prod: https:// api.tupu360.com/webservice/hrs/refuseCandidateInfoList
Beta: https:// betaapi.tupu360.com/webservice/hrs/refuseCandidateInfoList

企业编码（ECode）以及企业名称别名（EName），由TUPU提供
技术规范
服务的调用方通过标准的HTTPS协议跟服务通信, 服务返回的数据格式是标准的JSON。
客户端需支持TLS1.1及以上，如采用Java语言需使用JDK8。
接口调用限制
所有HTTPS请求，每个企业每天最多支持10000次，并发最多支持30个请求
Token生成规则：
接口调用必须接收有效的token参数。
Token 由Base64.encode(EName +@@+MD5.encode(secret)) 生成
EName由图谱提供。
Secret由ECode+new Date().format("yyyyMMdd")) 生成。
示例如下：

接口访问方式：http post请求
请求拒绝候选人信息
URL: 
/webservice/hrs/refuseCandidateInfoList
Post请求body（按时间范围请求）
Token:授权参数[请求时需要在request header中包含参数，不包含在post参数中]-生成规则
json格式数据
{
beginDate:'2017-07-01'//开始日期 从00:00:00算起
endDate:'2017-07-10'//结束日期 到23:59:59 [开始与结束日期间隔不可超过10天]
pageSize: '10' //分页每页多少数据，最多不能超过100条，字符串格式传递
pageNum: '1' //分页第几页，字符串格式传递
}

注:其中token为必须参数，beginDate、endDate非必填，默认查询最近十天发生状态变更的数据，并且结束日期与开始日期间隔不能大于十天


请求示例：
{

    "endDate": "2021-08-21",
    "pageSize": "10",
    "pageNum": "1"
}
