## Document Content

图谱招聘系统
获取指定状态下候选人信息
接口文档

前提条件
访问职位接口的服务器IP需加入白名单许可，请确认当前调用的机器为约定IP，并可访问
Prod: https://api.tupu360.com/webservice/hrs/…
Beta: https://betaapi.tupu360.com/webservice/hrs/

企业编码（ECode）以及企业名称别名（EName），由TUPU提供
技术规范
服务的调用方通过标准的HTTPS协议跟服务通信, 服务返回的数据格式是标准的JSON。
客户端需支持TLS1.2及以上，如采用Java语言需使用JDK8。
接口调用限制
所有HTTPS请求，每个企业每天最多支持10000次，并发最多支持5个请求

接口访问方式：http post请求
请求候选人信息
URL: 
/webservice/hrs/candidateInfoList
Post请求body（按时间范围请求）
token:授权参数[请求时需要在request header中包含参数，不包含在post参数中]-生成规则
json格式数据
{
"beginDate": "2021-07-01",//开始日期
"endDate": "2021-07-06",//结束日期[开始与结束日期间隔不可超过10天]
"status": "3",3:Offer阶段;4:Onboard阶段,候选人的招聘状态
"sub_status": "OF4"//候选人招聘状态下的小状态，如不填则返回所有小状态下的候选人，代码状态：根据客户配置决定，提供接口时再提供
"pageSize": '100' //分页每页多少数据，最多不能超过100，字符串格式传递
"pageNum": '1' //分页第几页，字符串格式传递

}
注:其中token为必须参数，beginDate、endDate非必填，默认查询最近十天发生状态变更的数据，并且结束日期与开始日期间隔不能大于十天
Post请求body（按单个申请cId请求，注:每个候选人对于每个职位有单独的cId）
token:授权参数[请求时需要在request header中包含参数，不包含在post参数中]-生成规则
json格式数据
{
"cId": 123456
}
