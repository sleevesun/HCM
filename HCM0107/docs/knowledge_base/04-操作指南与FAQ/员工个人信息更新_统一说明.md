## Document Content

一、主流程
员工打开“个人信息更新”，先出现照片页



进入个人信息页面，并根据不同情况，显示不同文案

每个模块点击“确认并保存”后，标签更新为“√（P）”，按钮变更为“查看”
点击个人信息页面的“提交”，校验是否四个模块都=√（P），或者=“提交成功（Y）”
如校验不通过，提示如下，有几个 ！=√（P）则提示几行

如校验通过，则更展示弹窗，勾选承诺书与告知声明，必须两个都勾选，才可点击“确认”

提交成功后
教育经历状态更新为“待审核（D）”，标签“待审核”，查看按钮隐藏
其他模块状态更新为“提交成功（Y）”，标签“无”，查看按钮可见
此时，总提交按钮不可见
提交成功后，待办清空
等待教育经历审核
如审核通过，状态更新为“提交成功（Y）”，标签“无”，查看按钮可见
如审核不通过，状态更新为“（U）”，标签“请确认”，按钮“确认”
此时，增加教育经历拒绝待办，点击后进去教育经历详情页，并可在该页面直接提交（提交按钮初始禁用，编辑一次后启用）

工作经历
保存/加载时，按照入学时间，从近到远排序。
3.1 每条校验
开始日期>=职业生涯开始日期
报错提示，开始日期应大于等于职业生涯开始日期
（2024-08-14在员工个人信息更新去掉，原因：用户测试阶段，有人记不住自己的职业生涯开始日期，其实就差几天，但也无法通过）
结束日期 < 最近一次入职日期    且   结束日期 > 开始日期
报错提示：离职时间日期应早于入职公司时间 /  离职时间应晚于入职时间
报错提示：结束时间应早于入职公司时间  /  结束时间应晚于开始时间

3.2 全体校验
最近一次入职日期之前，必须有一条类型=工作经历
报错提示：入职公司前的最近一段工作经历必须填写
（入职登记表）报错提示：工作经历不能为空
每一段之间间隔不超过6个月，包括最近一次入职日期与最后一条
报错提示：相邻经历之间间隔不能超过6个月，请补充实际情况
报错提示：最近一条经历与入职日期之间间隔不能超过6个月，请补充实际情况
（入职登记表）报错提示：最后一条经历与职业生涯开始日期间隔不能超过6个月，请补充实际情况
交时校验：
报错提示：各段经历之间时间不能重叠，请修改
（入职登记表）填写完整
先判断，满足>=3段工作经历（类型=工作经历），且，最远一条（不分类型）的开始日期至最近一条的结束日期，相差>=7年（用年月日计算）。
如不满足，继续判断是否 [ 职业生涯开始日期、入职日期 ] 之间，已全部填满（段与段之间不超过6个月，最近一次入职日期与最后一条，职业生涯开始日期与第一条）
如仍不满足，报错提示：请填写至少3段工作经历，如最近3段工作经历相加少于7年，则继续填写至不少于7年。如您毕业后工作经历少于3段，则全部填写。
