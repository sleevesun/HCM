1. 需求背景
为帮助提高绩效考核过程的系统使用体验，以及配合绩效管理层层拆解的管理
需求，因此绩效系统做了全新的设计及升级。
2. 需求内容
2.1 需求范围
绩效系统包括绩效绩效的填写、分解及审核；绩效最终的评分及审核；历史绩
效查询；绩效报表查看等主要功能。
绩效期次包括：年度绩效、季度绩效、月度绩效及半年度绩效。（本需求内的
规则暂时适用于所有类型的绩效期次。）
2.2 业务说明
2.2.1 绩效考评流程
参见《绩效系统流程图》PDF文档。
2.2.2 绩效评分审批流程
⚫ 游戏业务（COO Office）绩效考核审批流程：
员工→五级部门负责人→四级部门负责人→三级部门负责人→
HRD→HRA→二级部门负责人→HRVP→COO→上市公司总裁→集团CEO
→董事长。
⚫ 集团职能绩效考核审批流程：
员工→ 五级部门负责人→四级部门负责人→三级部门负责人→
HRD→HRA→二级部门负责人→HRVP→上市公司总裁→集团 CEO→董事
长。

⚫ 分子公司：
PA、未来教育、完美通识：
员工→五级部门负责人→四级部门负责人→三级部门负责人→
HRD→HRA→二级部门负责人→HRVP→董事长。
鲲鹏、完美世界教育
员工→五级部门负责人→四级部门负责人→三级部门负责人→
HRD→HRA→二级部门负责人→HRVP→集团CEO→董事长。
⚫ 补充说明：
以上流程中，没有该级别负责人的，直接到上一级审批人。
例如：没有五级部门负责人，直接到四级部门负责人进行审批。
2.2.3 绩效等级及分布规则
绩效等级及分数对应关系：
等级 分数范围（X）
S X≥120分
A 110≤X＜120分
B 100≤X＜110分
C 80≤X＜100分
D X＜80分
不同等级对应的分布比例（建议）：
等级 比例（年度）
S 10%
A 20%
B 55%
C 10%
D 5%
说明：验证绩效分布时，只需验证 S 级与 A 级的总人数是否符合要求
（=roundup（S+A的人数/总人数，0））。

3. 页面功能详情
3.1 菜单
界面：
说明：
⚫ 头像、员工姓名、员工ID：获取员工在ps系统中的信息；
⚫ 我的绩效：点击进入我的绩效页；有待操作任务时右上角显示小红点；
⚫ 绩效审批：有审批权限的员工可看到此操作，点击进入绩效审批页；有待审批
任务时右上角显示小红点；
⚫ 绩效报表：HRG、HRD、绩效负责人可看到此操作，点击进入绩效报表查看
页；
⚫ 设置：部门负责人、HRG、HRD、绩效负责人可看到此操作，点击进入绩效设
置页。
3.2 我的绩效
界面：

说明：
⚫ 当前绩效：
显示当前用户需参与的、未走完流程的绩效 *当期绩效开始时（自然时间），
且上期绩效已完成时显示；
【提交绩效计划】：绩效状态为“制定绩效”时的初始操作，点击进入绩效详
情页制定绩效；如有待接收的任务则在按钮右上角显示“需接收任务”；
【重新提交绩效】：绩效状态为“制定绩效”，且绩效计划被上级退回时显示
此操作，此操作与“提交绩效计划”不在同一绩效中同时出现，点击进入绩效
详情页制定绩效；
【接收绩效任务】：绩效状态为“制定绩效”时，如有上级分配绩效任务且未
接受时显示此操作，点击进入绩效详情页，弹出框提示接收绩效任务；
【查看绩效详情】：绩效状态为“绩效审核”、“绩效执行”、“绩效自评”、
“上级评分”时显示此操作，点击进入绩效详情页；

【分解绩效任务】：绩效状态为“绩效分解”时显示此操作，只有有下属的用
户有此状态及操作。
⚫ 历史绩效：
状态为“已完成”时的绩效在此显示，默认显示离当前时间最近的绩效入口，
最多显示5个，点击进入该绩效详情页；
5个绩效以上时显示“查看更多>>”，点击进入历史绩效查找页。
⚫ 完美世界【行为准则】：
静态显示行为准则内容。
3.3 绩效详情页
3.3.1 绩效状态整体说明
⚫ 无直接下属的用户：制定绩效>审核绩效>绩效执行>绩效自评>上级评分>完
成
⚫ 有直接下属的用户：制定绩效>审核绩效>绩效分解>绩效执行>绩效自评>上
级评分>完成
3.3.2 制定绩效
界面：

说明：
⚫ 默认显示5条绩效指标填写表单；
⚫ 每项绩效指标都包含维度、权重、指标、指标详情：
 维度选择：必选，业务指标、财务指标、团队发展、过程指标、个人发
展；
 权重：必填，只能填整数；
 绩效指标：必填，选择指标或自定义指标；
 绩效详情：必填，10-300字。
⚫ 绩效指标可删除和增加；
⚫ 【提交绩效计划】：提交时权重必须等于 100%，否则提示“您的绩效权重
不是100%，请您修改后再提交”；提交时必填项不能为空，否则提示“xxx
不能为空，请您修改后再提交。”；

⚫ 权重实时计算；
⚫ 填写内容实时保存。
3.3.3 绩效审核中
⚫ 绩效审核中时只能查看绩效详情；
⚫ 如绩效审批不通过则回到绩效制定中，并显示退回原因。
3.3.4 绩效分解
⚫ 绩效审核通过后，如有直接下属，则进入绩效分解状态；
⚫ 当所有直接下属的绩效都审批结束后，结束绩效分解状态，进入绩效执行中；
⚫ 绩效分解后可在该条绩效下点击查看绩效分解记录。
⚫ 绩效分解界面：
 财务指标分解：
弹出框内内容均为必填项；
当所选下属已提交绩效计划时，显示提示“此下属的绩效已经提交，您需
要先退回其绩效才能给其分配绩效。”；
 非财务指标分解：

弹出框内内容均为必填项；
当所选下属已提交绩效计划时，显示提示“此下属的绩效已经提交，您需
要先退回其绩效才能给其分配绩效。”。
3.3.5 绩效执行
⚫ 绩效执行中可查看绩效；
⚫ 年度绩效结束前1个月（即12月1日开始）可以点击“开启绩效自评”；
⚫ 季度绩效结束前2周可以点击“开启绩效自评”。
3.3.6 绩效自评
界面：

说明：
⚫ 可查看自己的月报，点击进入月报系统查看；
⚫ 每项绩效指标需填写实际完成情况及评分；
⚫ 评分为0-150的整数；
⚫ 整体绩效分数由权重得出，显示在右上角；
⚫ 绩效等级及分数对应关系：
等级 分数范围（X）
S 120≤X≤150分
A 110≤X＜120分
B 100≤X＜110分
C 80≤X＜100分
D X＜80分
3.3.7 上级评分

⚫ 上级评分中可查看自己的绩效详情；
⚫ 上级评分中只显示自评的分数。
3.3.8 完成
⚫ 绩效评分审批结束后，绩效考评完成，点击可以查看绩效详情及最终的绩效
等级。
3.4 绩效审批
3.4.1 审批绩效计划
界面：
说明：
⚫ 绩效审批分两个环节：审批绩效计划和审批绩效评分，每期绩效需要先完成绩
效审批，再完成绩效评分审批。有待审批任务时在按钮右上角显示数字。
⚫ 审批绩效计划：
绩效计划仅需直接上级审批；
审批及退回流程规则见文档《绩效系统流程图》；
左侧显示当前需要审批的绩效计划；
右侧显示选定的某一期绩效列表：默认显示“待审批”状态下的绩效列表；

⚫ 【待审批】：指下属已提交绩效计划，上级未审批的状态，可操作审批通过或
退回，操作退回时需填写原因，退回原因不得少于10个字；
⚫ 【已审批】：记录已经审批过的绩效，可操作查看绩效详情；
⚫ 【未提交】：指未提交绩效计划的状态；
⚫ 绩效考评人数：审批人所负责的参与某期绩效考评的总人数。

见审批及退回流程规则见文档《绩效系统流程图》；

⚫ 【未提交自评】：审批人所负责的参与某期绩效考评的人中，还未到提交自评
的记录（直接下属与所有下属分别计数）；

3.5 绩效报表
界面：

说明：
⚫ 选择查询条件：
 绩效选择：必选；
 部门选择：必选；三级/四级部门选项需根据上一级部门所选内容变化；
 点击【查找】后显示查询结果；
⚫ 报表可以以Excel形式下载，下载内容为筛选条件下的所有数据；
⚫ 绩效考评进度：显示所选绩效当前的考评状态；
⚫ 绩效分布：显示所选绩效当前的绩效分布；
⚫ 页面支持分页：每页15条数据。
3.6 设置
3.6.1 参与人员设置
⚫ 筛选条件：根据当前用户所有的部门权限范围选择；
⚫ 部门负责人可选择其所负责的部门；
⚫ HRD可选择其作为部门负责人所负责的部门，以及作为HRD所负责的部门；
⚫ 选择绩效后，设置参与绩效考评的人员，当所选人员为试用期员工时需弹出提
示：“员工xxx还在试用期，您确定把其设置为参与绩效考核吗？”；

⚫ 人员数据取自PS系统，不取实习生数据。
3.6.2 绩效目标导入
⚫ 以表格形式导入员工绩效，导入的绩效初始状态为“绩效分解”；
⚫ 导入的数据内容参考原型。
3.6.3 权限管理
⚫ 取PS系统权限，绩效系统中不单独设置。
⚫ 权限划分：
 部门负责人：审批权限；根据组织架构取其下属；人员设置权限；
 HRD：HR审批权限、报表查看权限；人员设置、绩效目标导入权限；
 HRG：报表查看权限、人员设置、绩效目标导入权限；
 绩效负责人：所有权限。

总裁、董事长；
