# 工薪预算 TAB - 部门级预算（离职补偿金、加班费、签约金）视觉展示方案

## 一、 需求背景与目标

在当前的“工薪预算”TAB中，除常规的 HC 工资、社保等基于人头的成本外，还需增加**部门级预算**（不挂载在具体人员身上的公共费用）的实时监控。
本次新增展示的科目包含：**离职补偿金**、**加班费**、**签约金**。
针对这 3 个科目，需分别展示其生命周期中的 4 个核心数据节点：

1. **年初预算**（基准）
2. **当前预算**（调整后）
3. **已用金额**（实际消耗）
4. **剩余金额**（当前预算 - 已用金额）

## 二、 全局交互与数据格式规范

在输出具体的视觉布局前，确立以下全局通用的交互与数据规范：

* **显示单位（极简原则）**：界面默认统一以\*\*“万元”\*\*为单位展示（保留1位小数，如 `12.5 万`），以降低长串数字对视线的干扰。

* **悬浮明细（Hover Tooltip）**：当鼠标悬停（Hover）在任意“万元”数值上时，通过黑色半透明的 Tooltip 弹出**精确到 0.1 元**的具体数值（如：`125,456.7 元`），满足财务级的对账诉求。

* **超支预警（弱预警）**：遵循 B 端界面的克制设计，不增加花哨的进度条。当某科目的\*\*“剩余金额” < 0\*\* 时，仅将该剩余金额的**数字文本标红**（如 <span style="color:red">-2.5 万</span>），反之则保持默认的深灰色。

***

## 三、 视觉布局设计（双方案）

结合当前的页面架构，提供以下两种视觉布局方案供业务评审：

### 方案一：新增独立的“部门级预算”TAB（推荐）

**1. 设计思路**
将部门级预算从“基于人头的明细表格”中彻底抽离出来，在现有的“工薪预算”TAB 的右侧，新增一个平级的 **【部门级预算】** TAB 页签。这种设计能够在不干扰底层 HC 数据结构的前提下，为财务提供清晰的科目管控视图。

**2. 视觉形态**

* **顶层交互**：在页面核心数据区下方，新增类似 `[ 工薪预算 ] [ 部门级预算 ]` 的 Radio/Tabs 切换控件。
* **UI 风格完全复用**：新 TAB 的表格组件完全复用原“工薪预算”主表格的 UI 风格，包括但不限于：
  * 表格列宽、行高、边框样式
  * 表头背景色（深灰色背景 `bg-gray-100` 或 `bg-gray-50`）
  * 斑马纹效果（`hover:bg-blue-50` / `hover:bg-gray-50`）
  * 空数据展示及后续可能的分页器样式
* **切换体验**：确保 TAB 切换时数据加载状态平滑，避免页面跳闪。

**3. 视觉草图 (Mockup)**

```text
[ 工薪预算 ]  [ 部门级预算 ] (当前选中)
──────────────────────────────────────────────────────────────────
| 使用情况     | 离职补偿金(万) | 加班费(万)   | 签约金(万)       |
| :---         | :---           | :---         | :---             |
| 年初预算     |      15.0      |      8.0     |       3.0        |
| 当前预算     |      15.0      |     10.0     |       3.0        |
| 已用金额     |       5.2      |     11.5     |       0.0        |
| 剩余金额     |       9.8      |     -1.5(红) |       3.0        |
──────────────────────────────────────────────────────────────────
```

*(注：鼠标移入表格内的任意数字，如* *`5.2`，即弹出 Tooltip 显示* *`52,345.0 元`)*

**4. 方案优劣**

* **优点**：逻辑最为清晰，信息架构极其严谨。彻底分离了“公共池”和“个体成本”，财务核算时可以直接在专门的 TAB 下查看全局的部门级花销，且扩展性极强（未来若增加“招聘费”等其他部门级科目，只需在该 TAB 下横向扩展即可）。
* **缺点**：增加了一次点击（Click）的交互成本。

***

### 方案二：汇总表格独立列横向扩充

**1. 设计思路**
按照“工薪预算”的主表格逻辑，将这 3 个科目直接融入到下方的数据网格中。具体定位在 **“正编”** 组件的数据行中，作为独立的列，插在 **“2026年全年”** 汇总列的**前面**。

**2. 视觉形态（应对 4 个指标的空间挑战）**
因为每个科目有 4 个数据（年初、当前、已用、剩余），如果直接平铺会导致表格横向无限拉长，极度影响阅读。因此采用\*\*“主副数据结合 + Tooltip”\*\*的降维展示法：

* **表头**：独立展示为“离职补偿金”、“加班费”、“签约金”三列。

* **单元格内容（主）**：单元格内直接展示最核心的\*\*“剩余金额”\*\*（超支则标红）。

* **单元格内容（副）**：在剩余金额下方，用极小的灰色字体附带\*\*“已用金额”\*\*。

* **Hover 气泡**：鼠标移入单元格时，Tooltip 完整展示该科目的 4 项明细数值。

**3. 视觉草图 (Mockup)**

```text
| 部门/人员 | ... | 离职补偿金(万) | 加班费(万)   | 签约金(万)  | 2026年全年总计 |
| :---      | :-- | :---           | :---         | :---        | :---           |
| 汇总：正编| ... | 剩: 9.8        | 剩: -1.5 (红)| 剩: 3.0     | 3,500.0 万     |
|           |     | (已用: 5.2)    | (已用: 11.5) | (已用: 0.0) |                |
```

*Hover 在「加班费」单元格时弹出的 Tooltip：*

> **加班费明细**
> 年初预算：80,000.0 元
> 当前预算：100,000.0 元
> 已用金额：115,000.0 元
> 剩余金额：-15,000.0 元

**4. 方案优劣**

* **优点**：空间利用率极高，不需要在顶部额外增加卡片区块，所有的成本数据都在一张大表中闭环。

* **缺点**：逻辑归属上略显勉强（离职补偿金等属于部门公共池，硬塞在“正编”这一行中，可能会让 HRBP 误以为这些费用是仅针对正编人员的）；且隐藏了“年初/当前”预算，必须通过 Hover 才能看到全貌。

***

## 四、 架构师建议

从 B 端产品的**信息架构严谨性**出发，我个人**强烈推荐方案一**。
原因在于：加班费、离职补偿金是**部门级公共池**的概念，而表格里的“正编”通常是基于具体 HC 编制聚合的成本。将公共池的钱强行塞入正编的列中，会导致财务逻辑上的混淆。方案一将其作为平级的独立 TAB 展示，既符合宏观监控的定位，也能彻底解耦人员成本与部门公共费用。
