# 02_业务流程说明

本篇档详细梳理了在 HCM (HC Planning 3.0) 系统与相关外围系统（如 OA、PS）联动时，涉及人员入职、离职、调动及调薪等核心业务场景下的流程流转、HC（Headcount）管控及预算校验规则。

## 1. 入职流程：HC 展示及控制

系统在人员入职发起及审批环节，对 HC 和工资预算进行强管控。

### 1.1 校验规则
在 OA 发起入职流程时，会调用 PS/HCM 接口获取该部门入职当月的 HC 及预算信息。
- **正式员工**：校验预计入职季度的季度末节点，同时校验 **HC 数量**及**工资预算**是否充足。
- **实习生**：校验预计入职的月度节点，**仅校验预算**是否充足。
- **数据依据**：按照 OA 中 HC 管理员维护的部门颗粒度取数（包含预算 HC、过渡性替换 HC 等）。

### 1.2 页面展示与流程管控
1. **发起校验**：
   - 用户选择部门及人员类型（如：正式/实习/外包）后，系统验证是否有空余 HC 或离职替换 HC。
   - 若无空余且无替换 HC，系统将阻断发起，并提示：“*xxx部门已无正式员工空余或替换编制，不能发起新的入职。*”
2. **提交校验**：
   - 在入职流程正式提交时，系统会**再次验证**空余 HC 状态，确保在并发操作时不会超额占用。
3. **HC 编码占用与释放**：
   - **占用**：发起入职流程时，系统需接入 OA 生成的 HC 编码并占用该编码，同时在途入职数 +1，空余 HC -1。
   - **作废/释放**：若入职流程终止或未入职，该编码作废，在途入职数 -1，空余 HC +1。
   - **入职成功**：读取 PS 入职信息后，在途入职数 -1（实体人员落地）。

## 2. 离职流程：替换 HC 释放

为保证业务运转的连续性，离职流程中引入了“替换 HC”概念。

### 2.1 流程控制点
- 所有的体内离职流程，均增加 **HR 负责人审批节点**。
- **提前释放**：离职申请在 HR 负责人审批通过后，系统即可提前释放 1 个“离职替换 HC”并传给 OA，用于发起新人的入职。

### 2.2 落地结算逻辑
- OA 读取 PS 的最终离职信息。
- **若真正离职**：离职替换 HC 数 -1，部门的正式空余 HC +1。
- **若未离职（撤销）**：离职替换 HC 数直接 -1（收回提前释放的额度），不增加空余 HC。

## 3. 调动流程：HC 及预算转移

针对人员在组织内部的调转，系统区分是否携带 HC 与预算：

### 3.1 不带 HC 及预算的调动
- 发起调动流程时，系统必须验证“**调入部门**”是否有足够的空余 HC 来接收该调入人员。
- 若调入部门无空余 HC，则不可发起该调动流程。

### 3.2 带 HC 及预算的调动
- 调动生效后，系统自动执行：**调出部门 HC -1，调入部门 HC +1**。
- 调动相关数据（调动时间、人员信息、是否带 HC 调动、当前状态）会实时同步传给 OA。

## 4. 组织变动场景的预算流转

当企业发生部门合并、拆分或撤销时，系统提供标准化的划转机制。

### 4.1 场景A：组织剪切
- 直接将整个组织节点及其下属人员、预算平移至新的父级节点下，各项额度保持不变。

### 4.2 场景B：撤销并承接（例：甲工作室策划部 -> 乙工作室）
1. **整体打包划转**：
   - 部门 ID 不变，该部门下所有在职人员、空余 HC、部门级预算全部自动迁移至新架构下。
2. **分步划转（部分人员或预算转移）**：
   - **Step 1**：人员通过“带 HC 调动”功能进行批量划转至新部门。
   - **Step 2**：系统根据配置，将原部门剩余的空 HC 和部门预算余额，通过“移余额”操作划转给新部门。

## 5. 调薪流程控制

调薪直接影响工薪预算，系统在此环节实现了全自动化闭环与前置校验。

1. **调薪前置预算校验**：发起阶段实时校验部门预算，若部门预算不足，无法发起调薪流程。
2. **在途流程预算占用**：审批中的调薪申请同样计入预算占用，防止并发调薪流程重复使用或超支预算。
3. **生效规则与自动追溯**：
   - 生效时间与发薪规则系统联动。
   - 若出现跨期生效，追溯金额由系统自动计算与补发，不再依赖人工判断或线下处理。
4. **调薪结果自动落地**：
   - 调薪在审批通过后即刻生效，系统自动流转并更新薪酬主数据。
   - 收敛权限：剥离 HRSSC 的薪酬主数据手工编辑权限，全流程在线留痕、可追溯。
