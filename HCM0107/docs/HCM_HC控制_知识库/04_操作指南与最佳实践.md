# 04_操作指南与最佳实践

本指南为 HRSSC、C&B、HRBP 等角色提供了在 HCM（HC Planning 3.0）系统上线前的必备配置检查清单（CheckList），以及日常业务操作中的最佳实践建议，以确保系统的数据准确性与预算管控的有效性。

## 1. 上线前配置 CheckList

为保证系统平稳上线并准确承接预算编制，各业务模块需在上线前 5 天至上线前完成以下存量数据的 Review 与清理。

### 1.1 组织架构与管理者核对
- [ ] **高管部门层级调整**：当前部分高管被挂在 5 级子部门下，需调整回“集团高管办公室”进行统一管控（责任方：HRSSC 非高管薪资组 / 高管薪资组）。
- [ ] **各部门 HRD 维护复核**：以集团办公室为例，需确保内审部、全球泛娱乐、高招办等部门的 HRD 设置准确无误，因为 HRD 是部门级预算的默认维护方（责任方：李云飞等）。

### 1.2 特殊人员类型数据审查
- [ ] **劳务派遣与兼职存量信息**：
  - 检查系统内维护的员工类型是否正确。
  - 核对薪资标准是否维护准确，需根据实际业务情况更新最新标准（责任方：HRSSC）。
- [ ] **人力外包薪资标准**：
  - 系统内的人力外包薪资标准必须是人员的“**总包成本**”。
  - 需要采购部配合核对系统数据是否与实际合同约定的总包成本一致（责任方：采购部）。
- [ ] **海外发薪人员数据**：审查境内外发薪人员的科目映射与成本归属是否正确（责任方：HRSSC / 高管组）。

### 1.3 系统基础配置与财务联动
- [ ] **部门“合同签署公司”字段维护**：
  - 重点核查 PS 系统中各个部门（尤其是新建部门，如火星工作室）是否维护了“公司”数据。若未维护，会导致 HCM 读取失败，从而无法进行预算编制（责任方：财务 / HRSSC）。
- [ ] **明确预算管控部门层级**：
  - 确认哪些部门需要管控到“预算汇总部门的 -1 层”，哪些只需管控到“预算汇总部门”（责任方：C&B）。

## 2. 日常操作最佳实践

### 2.1 新建部门时的“必做动作”
在进行组织架构调整或新建工作室时，除了在系统中创建部门节点外，**必须**同步在 PS 系统中为该部门维护“合同签署公司”字段。
> **注意**：哪怕该部门暂时不需要编制本年度预算，也应提前维护，否则未来一旦有人员调入或产生费用，系统将因缺失公司主体而阻断预算流程。

### 2.2 离职与入职的时间差管理
- **合理利用“替换 HC”**：由于系统管控严格，当部门 HC 满编但有员工即将离职时，不要等待员工彻底走完离职手续再发起新员工入职。
- **最佳路径**：让离职员工尽早发起离职流程，只要 HR 负责人（HRD）审批通过，系统就会立即释放 1 个“离职替换 HC”给 OA，此时即可顺利为新员工发起入职流程，保障业务不中断。

### 2.3 组织撤销与人员转移的平滑过渡
当部门被撤销并入其他部门时，建议采用**“分两步走”**的策略：
1. **先走人员调动**：通过“带 HC 调动”功能，将所有在职人员连同他们的 HC 和预算批量划转至新部门。
2. **再转移余额**：最后通过“移余额”功能，将原部门剩余的“空 HC”和“部门级预算余额”一次性划给新部门，确保旧部门彻底清零并关停。

### 2.4 调薪流程的权限与追溯
- **不再手工干预**：调薪流程审批通过后，系统会自动流转并更新薪酬主数据，HRSSC 无需再进行线下 Excel 的复制粘贴或手工改薪，所有操作务必在线上完成以保证审计留痕。
- **跨期调薪处理**：遇到跨月/跨期生效的调薪，只需在系统中正确设置生效时间，系统会自动计算历史追溯金额并补发，切勿通过“手工调整项”去补发差额，以免扰乱人工成本看板的数据统计。
