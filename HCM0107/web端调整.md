PC端工薪预算审批详情页重构 PRD 与开发指南

一、 核心架构总览

本页面采用 上下主从联动架构 进行降维重构。

上半部分（控制中枢）：顶部横向微型数据卡片组，承担全局导航与异动预警功能。

下半部分（数据画布）：多级部门明细数据表格，承担明细查验与对账功能。
整体设计要求高信噪比，通过动态隐藏无关数据列来提升阅读体验。

二、 顶部微型数据卡片组（控制中枢）

1. 业务逻辑与排序

顶部包含 7 个微型数据卡片，必须严格按照以下顺序横向排列：
部门整体 > 正编 > 人力外包 > 实习 > 劳务派遣 > 兼职 > 部门级预算。
(注：如果某类人员在数据源中 HC 和预算均为 0，则在实际渲染时直接隐藏该卡片。)

2. 卡片内容结构

每个人员分类卡片仅展示最重要的结论数据，包含 3 个元素：

标题与预警区：左上角为分类名称（如“正编”）；右上角为异动预警图标（预算增加显示红色向上箭头，预算减少显示绿色向下箭头，无变化显示灰色平横线）。

HC 最终值：格式为 HC：[申请后数值]，字重加粗。

预算 最终值：格式为 预算：[申请后数值]万，字重加粗。

特例处理（部门级预算卡片）：

标题保持常规黑色，不使用特殊颜色。

由于该分类不包含人员，因此不展示 HC 数据。为了保证与其他 6 个卡片视觉高度严格一致，原 HC 所在行必须使用 CSS 进行相同高度的透明占位。

3. UI 视觉与交互规范

横向布局：外层容器使用 Flex 布局，不允许换行。宽度溢出时允许横向滚动，但必须在 CSS 中美化或隐藏原生粗糙的滚动条。

默认状态：初始化时，默认选中第一个“部门整体”卡片。

状态切换：

选中态：浅蓝色底色 + 蓝色边框高亮，营造强烈的“当前焦点”暗示。

未选中态：纯白底色 + 浅灰色边框，悬停时边框微亮。

三、 下方明细数据表格（数据画布）

1. 标题与结构

表格顶部带有动态标题（格式：[当前选中卡片名称]明细数据），右侧附带红绿颜色的图例说明。

表格首列固定为“下级部门组织”，支持纵向滚动时吸附在左侧（Sticky），且带有明显的右侧分割线。

2. 表头与视觉降噪

采用三级复杂表头，且必须对“申请前、申请后、变化”进行严格的视觉分层：

申请前（弱化）：浅灰底色，常规字重，灰色字体。

申请后（核心高亮）：纯白底色，字重加粗，黑色字体，并带有轻微阴影凸显其“最终结果”的主权地位。

变化（预警）：浅灰底色，中等字重。数据本身必须带正负号，且严格应用语义色（增加为红色，减少为绿色，0为灰色）。

3. 动态视图联动逻辑（核心交互需求）

下方表格的结构必须根据顶部选中的卡片进行动态切换，规则如下：

场景 A：当选中【部门整体】或【正编】时

表格需展示完整的 3 大指标块：HC 配置、月度工薪预算 (万)、年度工薪预算 (万)。

场景 B：当选中【人力外包】、【实习】、【劳务派遣】、【兼职】时

动态降噪：必须在视图中动态隐藏 月度工薪预算 (万) 这一列（包含其对应的所有表头和数据单元格），仅保留 HC 和年度工薪，以节省屏幕空间。

场景 C：当选中【部门级预算】时

重构表格结构：完全隐藏上述的人员类指标表格，切换为一个专属的资金池表格。

该专属表格的指标块为：离职补偿金 (万)、加班费 (万)、签约金 (万)。

内部依然遵循“申请前、申请后、变化”的三列视觉降噪规范。

四、 前端与 Tailwind 框架落地建议

建议使用 Vue、React 等响应式框架的状态管理来控制卡片激活索引（Active Index）。

对于动态隐藏列（场景 B），建议在月度工薪的 <th> 和 <td> 节点上绑定特定的 class 或动态 v-if/{&&}，实现无缝显隐。

切换视图时，建议给表格容器添加一个 200ms 的透明度过渡动画（Fade In），缓解 DOM 结构突变带来的视觉生硬感。
