# 移动端工薪审批详情页：渐进展开式架构说明 (方案三 - 已落地版)

## 一、 核心设计理念

本方案全面摒弃了传统 B 端系统在移动端强行平铺“重度表格”的粗暴做法，深度应用了**渐进式披露 (Progressive Disclosure)** 的交互法则。

- **产品心智契合**：精准匹配审批人“先看总盘是否健康，再看个别异动类目，最后深挖明细数据”的下钻式决策路径。
- **极高的数据墨水占比**：在首屏收起状态下，战略性隐藏“申请前数据”与“月度工薪”等中间推演过程，将屏幕像素全部让渡给“最终审批结果”与“成本异动量”。
- **智能交互控制**：对于无实质性变化的类目，自动禁用展开功能，减少无效点击，引导决策者聚焦于有异动的风险项。

## 二、 页面信息架构与版块拆解

页面在垂直方向上分为三个核心逻辑层：**部门整体看板**、**人员工薪预算变化**、**部门级预算变化**。

### 1. 部门整体看板 (Overview Card)

- **产品定位**：全盘定调。让决策者在一秒内明确点击“通过”后对部门整体成本的最终影响。
- **UI 规范**：
  - **标题**：`部门整体` (H3, Bold)。
  - **布局**：白色卡片容器，内部为垂直列表。每行采用 Flex 两端对齐 (`justify-between`)。
  - **指标项**：
    1. `申请后总HC`
    2. `申请后总工薪预算(万)`
- **数据表达格式**：
  - **左侧**：指标名称（灰色）。
  - **右侧**：`[最终值] ([变化值])`。
  - **颜色逻辑**：变化值部分应用语义色（增红减绿）。

### 2. 人员工薪预算变化 (Personnel Budget Group)

- **产品定位**：按人员类别将数据进行沙盒化隔离。
- **分类排序**：
  1. 正编 (Regular)
  2. 人力外包 (Manpower Outsourcing)
  3. 实习 (Intern)
  4. 劳务派遣 (Labor Dispatch)
  5. 兼职 (Part-time)
- **数据展示策略**：
  - **正编**：展示 HC、年度工薪、月度工薪。**不展示**离职补偿金和加班费（已移至部门级预算）。
  - **其他分类**：仅展示 HC、年度工薪。**不展示**月度工薪。

### 3. 部门级预算变化 (Department Budget Group)

- **产品定位**：独立展示非人员维度的预算异动。
- **包含项目**：
  - 离职补偿金
  - 加班费
- **展示形式**：与人员分类保持一致的手风琴卡片样式。
- **摘要指标**：统一显示为 `申请后预算(万)`。

## 三、 手风琴组件内层UI与逻辑规范

### 1. 收起态：类目概览行 (Accordion Header)

- **布局拆解**：
  - **左侧锚点**：定宽且加粗的类目标题（如“人力外包”），作为视线左侧扫描的起始锚点。
  - **右侧高密度数据**：使用 Flex 垂直方向的 `column` 排列，堆叠展示该类目的核心异动。
  - **右侧箭头**：
    - **有变化**：显示向下的线性箭头图标 (Chevron)，支持点击展开。
    - **无变化**：隐藏箭头，且整行不可点击（Disabled）。
- **对齐规范**：
  - **标签 (Label)**：左对齐。
  - **数值 (Value)**：右对齐，确保多行数据的数字位对齐。
- **文案格式**：
  - `申请后[指标名]` (Label)
  - `[申请后数值] ([差异值])` (Value)

### 2. 展开态：双栏数据矩阵 (Accordion Panel)

当用户点击某类目展开后，内部采用“左键右值”的双栏堆叠列表模式。

- **左侧维度栏 (Metric Label)**：
  - 设定固定宽度（如 `96px`），垂直居中展示具体指标名称（如“HC”、“年度工薪(万)”）。
  - 字重加粗，使用深灰色。
- **右侧度量栏 (Metric Values)**：
  - 占据剩余全宽，左侧有灰色分割线。
  - 内部垂直堆叠 3 行数据：
    1. **申请前**：灰色，常规字重。
    2. **申请后**：深色，加粗。
    3. **变化**：带正负号，应用强烈的语义色（**增红减绿**）。

## 四、 交互与逻辑细节

### 1. 语义色规范 (Color Logic)
- **增加 (+)**：红色 (`text-red`)。用于预警成本上升。
- **减少 (-)**：绿色 (`text-green`)。用于表示成本节约。
- **持平 (0)**：灰色 (`text-gray-400`)。

### 2. 智能禁用逻辑 (Smart Disable)
- **判定规则**：如果某分类下的**所有**摘要指标（HC变化 和 工薪变化）的绝对值均为 0。
- **表现**：
  - 手风琴表头置灰或保持常规态但**不可点击**。
  - 右侧箭头隐藏。
  - 无法展开查看明细。

### 3. 数据拆分与过滤 (Data Filtering)
- **正编分类**：自动过滤掉 `离职补偿金` 和 `加班费` 指标，避免与“部门级预算”重复。
- **非正编分类**：自动过滤掉 `月度工薪` 指标，保持界面简洁。

## 五、 技术实现参考 (Vue 3 + TypeScript)

- **组件结构**：
  - `BudgetApprovalMobile.vue`: 主控组件，负责数据获取与计算属性转换。
  - `BudgetScenarioThreeView.vue`: 纯展示组件，负责手风琴渲染与样式。
- **数据流**：
  - 原始数据 (`BudgetApprovalMobileData`) -> 计算属性 (`scenarioThreeGroups`) -> 视图组件 (`props`)。
- **样式方案**：
  - Scoped CSS，使用 Flexbox 与 Grid 布局。
  - 动画：`max-height` + `transition` 实现手风琴平滑展开。
