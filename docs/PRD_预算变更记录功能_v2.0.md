# HCM 系统：预算变更记录功能全量设计方案 (v2.0)

## 1. 方案概述
### 1.1 背景描述
目前 HCM 系统的预算变更逻辑散落在各个业务模块（如：预算审批、人员调动、组织架构调整）。由于缺乏统一的变更日志，财务系统无法准确追溯每一笔人力成本变动的具体原因及对应的项目归属。这导致“人财对账”困难，项目成本核算不精准。

### 1.2 核心目标
1.  **全面追溯**：记录每一笔影响编制和预算的业务操作，确保审计透明。
2.  **财务对账**：提供明细到“财务科目”的变动数据，支撑财务系统入账。
3.  **多维管控**：确保所有预算变动均带有“项目标签”，支持按项目维度的成本核算。
4.  **自动化同步**：通过数据中台实现 HCM 日志向财务系统的自动流转。

---

## 2. 功能模块拆分
### 2.1 模块一：HCM 系统前端变更展示
- **目标用户**：人力资源负责人、业务部门经理。
- **展示入口**：工薪预算驾驶舱 -> 预实比对 -> 预算变更记录。
- **核心功能**：
    - **汇总列表**：展示日志编号、发生时间、变更类型、涉及部门、HC 变动数、总额变动。
    - **嵌套明细**：点击单行可展开，查看该笔变更拆分到 14 类财务科目的具体金额及对应的项目标签。
    - **查询过滤**：支持按部门、变更类型、时间范围进行多维度检索。

### 2.2 模块二：后台日志同步传输系统
- **目标用户**：系统管理员、财务系统集成方。
- **工作原理**：
    - **触发捕获**：当业务逻辑改变预算边界时，自动向 `hcm_budget_change_log` 表写入数据。
    - **数据资产化**：数据中台定时抽取日志表，生成标准资产 `dw.hcm_budget_change_log_snapshot`。
    - **财务订阅**：财务系统订阅中台资产，获取增量变更日志完成入账。

## 3. 预算科目定义
变更日志必须精准记录以下 14 类科目的变动明细：
1.  **工资预算**
2.  **绩效工资**
3.  **商业保险（补充医疗）**
4.  **工会经费**
5.  **饭补**
6.  **工伤保险**
7.  **医疗保险**
8.  **失业保险**
9.  **养老保险**
10. **公积金**
11. **签字费**
12. **离职补偿金**
13. **加班费**
14. **（其他预留科目）**

## 4. 业务逻辑与触发机制
### 4.1 产生日志的场景 (产生日志并传输)
系统仅在以下涉及“预算边界变更”的场景下记录并传输日志：
- **预算审批**：年度或期中审批单通过，直接增减部门的预算额度。
- **带 HC 调动**：人员连带编制跨部门转移，系统生成调出（负值）与调入（正值）的对冲日志。
- **部门撤销**：部门关闭导致空余编制失效，系统记录额度回收。
- **部门剪切**：部门在组织树上位置变动，记录预算快照的整体平移。

### 4.2 补差额场景处理逻辑 (不产生日志且不传输)
针对由系统自动触发的“补差额”逻辑，方案定义如下：
- **触发场景**：新员工入职、普通人员调动、个人调薪。
- **逻辑定义**：系统根据占用人员的实际工资重新计算 HC 预算，差额自动转移至该部门的“补差额 HC”。
- **项目标签处理**：支持“补差额 HC”的项目标签与原始 HC 标签不一致（例如原始 HC 归属项目 A，差额归属“通用项目”）。
- **排除规则**：此类变动属于既定额度内的“内部头寸对冲”，**不产生**前端展示日志，也**不向**财务系统传输。

---

## 5. 技术集成与非功能性需求
### 5.1 数据链路
- **落地路径**：HCM 实时落地 -> 数据中台资产 -> 财务系统获取。
- **一致性**：日志记录与业务操作必须满足数据库事务的原子性。

### 5.2 性能与安全
- **加载速度**：前端日志列表首屏加载时间应小于 1 秒。
- **数据安全**：仅具备预算查看权限的用户可访问变更记录。
- **审计留存**：日志数据永久保留，满足审计合规要求。

## 6. 验收标准
- [ ] 汇总列表能正确展示 HC 变动和总额变动。
- [ ] 点击展开能查看到细化到 14 类科目的金额及项目标签。
- [ ] 验证入职场景产生的补差额数据确实没有出现在日志列表中。
- [ ] 验证带 HC 调动场景下，调出与调入部门的日志金额完全对冲。
- [ ] 财务系统能通过数据中台资产获取到完整的明细记录。
