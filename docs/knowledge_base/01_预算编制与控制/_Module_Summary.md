# 01_预算编制与控制 - 核心知识总结

## 模块概述
本模块是“人财一体化”的核心，涵盖了 HCP 3.0 至今的预算编制、预算调整、管控校验机制以及系统测试用例。主要解决预算如何线上化、如何打标分类以及如何在源头上控制超编问题。

## 核心知识点提炼

### 1. 预算编制与签约金逻辑 (HCP 3.0)
- **核心逻辑**：明确了基础的预算编制流程，并特别增加了针对签约金（Sign-on Bonus）的处理和摊销逻辑。
- **标准化审批链路**：
  - **游戏 & 电竞**：`HRD` -> `C&B负责人` -> `二级部门负责人` -> `业务HRBP head` -> `业务HR负责人` -> `1.1部门负责人` -> `CEO & 集团HR负责人`。
  - **集团职能 & 影视**：`HRD` -> `C&B负责人` -> `二级部门负责人` -> `业务HR负责人` -> `1.1部门负责人` -> `集团HR负责人` -> `董事长`。
- *溯源文件*：[【PRD】HCM系统（人财一体_HCP+3.0_预算编制）.md](./【PRD】HCM系统（人财一体_HCP+3.0_预算编制）.md)

### 2. 预算调整与管控闭环 (HCM 3.1)
- **业务痛点**：过去 HC 申请走线下，过程无法追踪；缺乏紧急情况的调剂能力。
- **管控对象**：
  1. **HC 数量**：以二级管控部门为单位，控制各员工类型的编制数。
  2. **工资预算**：仅控制对应 HC 的工资，**不再控制**工薪成本（如社保、公积金、饭补等），与旧版 HCP 2.1 形成区别。
- *溯源文件*：[【PRD】HCM系统（人财一体_3.1_预算调整与管控）.md](./【PRD】HCM系统（人财一体_3.1_预算调整与管控）.md)

### 3. 年初标识与差异管控
- **“年初预算”打标**：为配合财务（TA）系统计算预算差异，HCM 需准确界定部门每年的“初始预算”。在审批单中加入明确的打标逻辑，用以区分日常调整和年度初始划拨。
- *溯源文件*：[【PRD】HCM_年初标识&管控校验&按钮优化（2月）.md](./【PRD】HCM_年初标识&管控校验&按钮优化（2月）.md)
