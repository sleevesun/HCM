# 05_系统切换与过渡期执行 - 核心知识总结

## 模块概述
本模块聚焦于从旧版 HCP（人力成本规划）系统向新版 HCM（人力预算管控）系统平滑迁移的过程，以及在此期间的业务过渡方案、时间节点和权限配置。

## 核心知识点提炼

### 1. HCM 与 HCP 系统对比与差异
- **数据源与定位差异**：在 2026 年预算的编制和执行中，HCP 更多是作为静态编制系统，而 HCM 需要承载动态执行和数据回溯。
- **1月份管控执行过渡**：因新旧系统在年初（如 1 月份）的衔接可能导致断层，需要将后台静态数据（或前序年份结转数据）从 HCM 导出后，加工导入至新一年的预算池，实现跨年的平滑切换。
- *溯源文件*：[HCM_HCP系统切换与过渡期执行方案.md](./HCM_HCP系统切换与过渡期执行方案.md)

### 2. 上线时间表 (Milestones)
- **2025/9/3**：完成与核心 HRD（人力资源总监）的业务宣贯与系统确认，开启 HRD 测试环境（UAT）。
- **2025/9/17**：正式生产环境（PROD）发布，系统管理员完成所有线上基础配置（如审批流、部门树、科目映射等）。
- **2025/9/18**：HRD 正式进入系统开始编制新一年的预算（初期暂无强卡点校验，以熟悉录入为主）。
- *溯源文件*：[HCM系统上线准备文档.md](./HCM系统上线准备文档.md)
