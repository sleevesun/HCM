# 02_入职与调动校验 - 核心知识总结

## 模块概述
此模块专注于解决“无预算，不发生”的问题，即在实际业务系统（如 OA 或 PS）发起人员入职、调动等准入环节时，如何与 HCM 及财务系统（TA）进行强强联动的预算扣减与校验逻辑。

## 核心知识点提炼

### 1. 预算双向校验机制 (HCM & TA 协同)
- **业务痛点**：财务系统 (TA) 需要掌握人力预算的编制及调整情况（尤其是当前预算与年初预算的差异）。HCM 需要从 TA 获取实时的已发薪成本，以防超预算发薪。
- **校验方案**：
  1. **HCM -> TA (单向推送)**：HCM 推送最新预算数、变动明细等，以支持财务在总账层面做宏观控制。
  2. **TA -> HCM (校验与阻断)**：在入职或调动时，调用接口对比当月实际已支出 + 预估支出是否超出预算池。
- *溯源文件*：[HCM-TA+工薪预算数据一致性校验方案.md](./HCM-TA+工薪预算数据一致性校验方案.md), [【会议纪要】HCM&TA工薪预算双向校验技术方案讨论.md](./【会议纪要】HCM&TA工薪预算双向校验技术方案讨论.md)

### 2. 准入环节强校验 (入职/调动)
- **业务流转**：当业务侧发起人员“准入”（如入职、调入）或“主动变更”（如调薪）时，系统必须调用 HCM 接口获取对应部门、对应人员类型的 HC 数量与工资预算额度。
- **失败阻断**：若剩余 HC 为 0 或剩余工资预算不足，流程将报错阻断，要求业务侧先完成预算申请或调整。
- *溯源文件*：[【需求沟通】HCM-TA+工薪预算数据交互.md](./【需求沟通】HCM-TA+工薪预算数据交互.md)
