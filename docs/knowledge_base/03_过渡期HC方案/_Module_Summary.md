# 03_过渡期HC方案 - 核心知识总结

## 模块概述
本模块解决的是在人力资源管理中常见的“青黄不接”或“新老交替”时期的人员编制（HC）管理问题，旨在提升业务调配灵活性，防止因死板的系统卡点导致业务受阻。

## 核心知识点提炼

### 1. “过渡期 HC” 业务场景与定义
- **问题背景**：当现有员工（A）即将离职，但业务急需新人（B）提前入职进行交接时，若该部门编制已满，新人（B）将因“超编”无法在系统走入职流程。
- **解决方案**：引入“过渡期 HC”概念，允许在特定条件下，部门**暂时性超编**。

### 2. 系统管控与校验规则
- **入口管控**：系统仍保留严格的校验，仅在“准入”（入职/调入）和“主动变更”（调薪）时触发校验。
- **历史遗留问题处理**：对于历史遗留的超编问题不进行追溯和拦截，但新发生的准入必须有合法的预算/HC 来源（或符合过渡期 HC 申请条件）。
- **审批与释放**：过渡期 HC 必须经过特殊审批流，且有明确的有效期（如 30 天或 60 天）。当老员工离职或有效期满后，系统需自动释放该过渡期 HC，恢复正常编制数。

*注：开发实施时，需重点关注“过渡期”的时间窗口调度任务，确保超期后的编制能够准确释放或触发警报。*
