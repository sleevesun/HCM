# HCM 系统：预算变更记录功能全量详细设计方案 (v4.4)

## 1. 方案背景与核心价值

### 1.1 业务背景
目前 HCM 系统的预算变更分布在审批、调动、组织调整等多个模块。为了实现“人财对账”的精准性，财务系统需要获取每一笔变更的明细数据（科目级 + 项目标签级）。本功能作为 HCM 系统与财务系统之间的“数据枢纽”，通过结构化的日志记录，追踪所有涉及编制（HC）与人力成本预算的业务变动。

### 1.2 核心价值
- **精细化追溯**：为 HR 提供直观的变更流水查询。通过移除冗余的“日志编号”与“科目 ID”，让非财务专业用户也能快速理解变动逻辑。
- **业务对冲透明化**：在调动及部门剪切场景下，通过“调入/调出”成对出现的日志，清晰呈现预算的流向。
- **自动化集成**：静默处理系统自动产生的补差额，确保同步至财务系统的每一笔数据都具有明确的业务来源。

## 2. 用户旅程与关键场景

### 2.1 典型用户旅程
1. **触发变更**：HR 在“人员调动”或“部门调整”模块发起业务流程。
2. **流程审批**：审批通过后，系统自动生成对冲日志（一增一减）。
3. **查询回溯**：用户在“预算变更记录”中按时间降序查看最新变动。
4. **明细下钻**：展开记录查看科目分摊。若业务涉及多个项目（如 5 人跨 3 项目调动），明细表将按“科目+项目”维度平铺展示。
5. **对账分析**：用户根据变动金额与项目标签，与部门实际预算执行情况进行对比。

### 2.2 关键业务场景
| 场景名称 | 业务逻辑 | 核心展示规范 |
| :--- | :--- | :--- |
| **带 HC 调动** | 跨部门对冲（TRANSFER_OUT / IN）。 | 分别生成调出、调入两条日志。 |
| **预算审批** | 增量/减量预算申请。 | 单条日志，展示变动总额。 |
| **部门调整** | 部门剪切、合并、撤销（DEPT_MOVE_OUT / IN）。 | **调出/调入**成对生成，体现预算平移。 |
| **补差额场景** | 系统自动重算补差额。 | **不生成日志**，UI 中不展示。 |

## 3. 前端交互与信息架构

### 3.1 界面原型与样式规范
- **降噪设计**：
    - 移除“日志编号”、“财务科目 ID”。
    - 移除顶部搜索框（保留业务类型筛选）。
    - 移除“TA 系统同步状态”等底层技术信息。
- **排序规则**：默认按**发生时间（timestamp）降序**排列。
- **视觉反馈**：
    - 数值增加：红色 `+` 前缀。
    - 数值减少：绿色 `-` 前缀。
    - 业务标签：调动（橙色）、审批（蓝色）、部门调整（紫色）。

### 3.2 汇总列表字段 (Master)
| 字段名称 | 说明 | 视觉规范 |
| :--- | :--- | :--- |
| 发生时间 | 业务生效的精确时间 | 降序排列 |
| 变更类型 | 调入、调出、审批、部门调整等 | Tag 区分颜色 |
| 涉及部门 | 发生变更的目标部门全称 | - |
| 类别 | 人员类别（正编等）或费用类别 | - |
| HC 变动 | 编制数量增减 | 红加绿减 |
| 总额变动 | 预算总金额变动（元） | 红加绿减 |

### 3.3 嵌套明细子表字段 (Detail)
支持**多项目平铺展示**逻辑：
| 字段名称 | 说明 | 示例 |
| :--- | :--- | :--- |
| 财务科目名称 | 14 类标准科目名称 | 工资预算 |
| 项目标签 | 变动归属的项目名称 | 项目 A、项目 B |
| 变动金额 | 该科目在该项目下的具体变动值 | +50,000.00 |

## 4. 后台系统与数据模型

### 4.1 核心数据表结构 (Refined)
#### 主日志表 (budget_change_log)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| biz_type | Enum | TRANSFER_OUT/IN, APPROVAL, DEPT_MOVE_OUT/IN |
| dept_id | String | 涉及部门代码 |
| amount_total | Decimal | 总变动金额 |
| effective_time | Datetime | 发生时间 |

#### 明细表 (budget_change_detail)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| log_id | String | 关联主日志 ID |
| account_name | String | 14 类科目名称 |
| project_tag | String | 项目标签 |
| amount_delta | Decimal | 明细金额（支持多项目平铺） |

---

## 5. 验收标准

1. **信息密度**：列表不再展示冗余的 ID 类字段，一眼即可看清业务变动。
2. **场景对冲**：部门剪切场景必须同时生成“调出”与“调入”两条记录，且金额对冲。
3. **多项目支持**：当一笔业务涉及多个项目标签时，明细子表需正确平铺展示所有项目项。
4. **静默过滤**：补差额操作严禁产生日志，确保变更记录仅对应主动业务操作。
